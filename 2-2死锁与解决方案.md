
死锁的定义
```
两个或多个线程相互等待对方持有的资源且不释放手中的资源，导致所有线程都无法继续执行，程序永久阻塞
如:线程A占有锁1并等待锁2，线程B占有锁2并等待锁1，两个线程不释放资源且相互等待，从而造成死锁现象
```

死锁的四大条件
```
1.互斥：资源一次只能被一个线程占用(互斥锁特性)
2.持有并等待:线程拥有至少一个资源并等待其他资源
3.不可剥夺:拥有的资源只能主动释放不能被强制剥夺
4.循环等待:线程之间相互等待对方的资源，形成闭环

四个条件缺一不可，只要破坏其中一个死锁就不会发生
```
---
常见死锁现象


1.顺序不一致的嵌套锁
```
mutex m1,m2;                         //定义互斥锁m1,m2

void thread1()
{
    m1.lock();
    for(int i=0;i<=1000000;i++);     //让线程2能拿到m2(延时制造死锁)
    m2.lock();                       //等待m2但是不释放m1
    
    m1.unlock();
    m2.unlock();
}

void thread2()
{
    m2.lock();
    for(int i=0;i<=100000;i++);      //让线程1能拿到m1(延时制造死锁)
    m1.lock();                       //等待m1但是不释放m2

    m2.unlock();              
    m1.unlock();
}
```

2.单线程死锁
```
mutex m1;                            //定义互斥锁m1

void thread1()
{
    m1.lock();                       //手动加锁
    cout<<"dead locking..."<<endl;
    m1.lock();                       //再次加锁，占有的同时请求，自等待产生死锁
    m1.unlock();
}
```

3.忘记或未解锁(提前返回、抛出异常等导致)
```
mutex m1;                             //定义互斥锁m1

void test1()
{
    m1.lock();
    cout<<"test1"<<endl;
    return;                           //提前返回导致m1未解锁
    m1.unlock();                 
}

void test2()
{
    m1.lock();                        //m1未解锁，导致t2阻塞
    cout<<"test2"<<endl;
    m1.unlock();
}
```
---
死锁的解决方案


1.手动同步多锁顺序，破坏等待条件
```
例如让线程1和线程2都先请求m1再请求m2，避免各拿一半的资源

mutex m1,m2;                          //定义互斥锁m1,m2

void test1()
{
    lock_guard<mutex> g1(m1);         //先拿m1
    lock_guard<mutex> g2(m2);         //拿到m1再拿m2
    cout<<"hello test1"<<endl;
}

void test2()
{
    lock_guard<mutex> g1(m1);         //先拿m1
    lock_guard<mutex> g2(m2);         //拿到m1再拿m2
    cout<<"hello test2"<<endl;
}
```

2.std::lock自动同步多锁
```
lock全局函数一次性加多个锁，顺序由函数保证，手动加锁需配合adopt_lock或defer_lock

原理：
其内部采用死锁避免算法，它自动同步多锁的顺序，然后依次尝试加锁，在尝试过程中若发现
所需锁被占用则会回退，释放手中所有的锁资源，并等待一段时间后再尝试，析构时按顺序依
次解锁，如果有多个线程调用std::lock，lock之间会自动协调(不断尝试加锁直到确定顺序)

注意事项：
std::lock的参数至少为两个，不可单锁
std::lock和手动锁混用不会产生死锁，但可能影响效率(不断回退直到全部加锁成功)
```

```
mutex m1,m2;                                     //定义互斥锁m1,m2

void test1()
{
	lock(m1,m2);                                 //由lock一次性加锁
    lock_guard<mutex> g1(m1,adopt_lock);         //接管m1锁
    lock_guard<mutex> g2(m2,adopt_lock);         //接管m2锁
    cout<<"hello test1"<<endl;
}

void test2()
{
    lock_guard<mutex> g1(m1,defer_lock);         //绑定m1锁，延迟加锁
    lock_guard<mutex> g2(m2,defer_lock);         //绑定m2锁，延迟加锁
    lock(m1,m2);                                 //由lock一次性加锁
    cout<<"hello test2"<<endl;
}
```

3.scoped_lock作用域锁
```
c++17 RAII模板类，自动推导mutex类型(无需模板参数)，且内部自动同步多锁(对std::lock+adopt_lock/defer_lock二次封装，简化了语法并且可以单锁)
```

| 成员方法 | 参数列表         | 备注                                |
| ---- | ------------ | --------------------------------- |
| 有参构造 | mutex m1,... | 构造并绑定多个锁(可单锁)，然后使用std::lock自动同步多锁 |
| 析构   | 无            | 析构并依次解锁                           |
```
mutex m1,m2;                                     //定义互斥锁m1,m2

void test1()
{
	scoped_lock sl(t1,t2);                       //绑定多个锁并同步顺序
	for (int i=0;i<100000;i++);
}

void test2()
{
	scoped_lock sl(t1, t2);                      //绑定多个锁并同步顺序
	for (int i= 0;i<100000;i++);
}
```
